Sql object provider (SOP)

C'est ici que l'implémentation des différentes méthodes d'accès propres à chaque objets d'affaire est faite. Le SOP s'occupe également de la transformation des objets d'affaire en un format compatible pour les différentes transactions faite sur la base de données par l'appel des fonctions d'accès du ADA. Le procédé inverse est également fait lors de la récupération des données dans la BD.

Fonctionnement

Dans le cas d'une insertion ou d'une modification, avant l'appel de la fonction du ADA approprié, une version map(clef,valeur) de l'objet d'affaire sera créé par l'appel de la méthode GetMapAvec'X'() pour préparé l'objet à être envoyer au niveau de la base de données.

Pour une suppression, un simple appel à la fonction effacerDonnees du ADA sera fait.

Dans le cas de récupération de données, la version map de l'objet retourné par la fonction de récupération des données du ADA sera alors traité par la fonction Get'X'AvecMap() pour nous retourné la version "objet d'affaire" construite.
