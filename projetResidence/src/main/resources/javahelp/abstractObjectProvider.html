Abstract object provider (AOP)

Cette classe permet de créer une couche d'abstraction entre les objets d'affaire et les fonctions d'accès à la base de données. La création d'une instance de cette classe est donc requise pour accéder au niveau des fonctions du SqlObjectProvider. Il faut également noté que c'est dans le AOP que les méthodes de gestion de connexion ainsi que la méthode gérant la création d'instance du dataSource sont implémentées pour le niveau du ADA.

Fonctionnement

Pour obtenir une connexion, il suffit de faire appel à la méthode getConnection() qui, par la création d'une instance du dataSource si elle n'est pas déjà créée, nous renvoie un objet de connexion à la base de données residenceCarrefourBD.

Pour fermer une connexion, la méthode freeConnection() s'occupera d'effectué cette tâche proprement.

Finalement, pour avoirs accès aux différentes méthodes abstraites implémentées dans le SOP, un objet AbstractObjectProvider sera créer lors de l'appel de la fonction getInstance() implémenté dans le SOP.
